# Цифровой радар Актюбинской области

Постоянная публичная версия: `https://kassenkhann-cell.github.io/uct_sdu/sdu_uct_analiz/`.
Она обновляется через GitHub Actions после успешной проверки опубликованного
набора данных. При ошибке сохраняется последняя рабочая версия.

Чтобы обновить данные, замените Excel в папке `data` и запустите
`Запустить дашборд.cmd`. Проверенные данные автоматически отправятся на постоянную
ссылку и появятся там примерно через минуту.

Управленческий BI-дашборд для мониторинга качества интернета и связи. Приложение сопоставляет инфраструктуру населённых пунктов с профильными обращениями Е-өтініш, формирует рейтинг районов и адресный план действий.

## Быстрый запуск

Требуется Node.js 20+.

Самый простой способ на Windows — дважды нажать:

```text
Запустить дашборд.cmd
```

Браузер откроется автоматически после сборки.

При следующих запусках дашборд откроется сразу по одной ссылке
`http://localhost:5173/sdu_uct_analiz`. Сервер работает в фоне, поэтому держать окно терминала
открытым не нужно. Повторный двойной клик просто откроет уже работающий дашборд.

Коллегам отправляйте постоянную публичную ссылку в начале этого файла. Она работает
с любого компьютера и не требует, чтобы ваш компьютер оставался включённым.

## Аналитик данных

Чат использует OpenAI-совместимый API `https://llm.nitec.kz/v1` и автоматически
выбирает модель DeepSeek V4 Pro. Локальный запуск безопасно читает ключ из
`data/.env`; значение ключа не попадает во frontend, логи или GitHub.

Район указывается прямо в вопросе. Например: `сравни Хромтауский и Байганинский
районы` или `подготовь отчёт по Мугалжарскому району`. Если район не указан,
аналитик использует данные всей области.

Для серверного размещения задайте `LLM_API_KEY` как секрет окружения и укажите
адрес backend в переменной сборки GitHub `VITE_API_BASE_URL`. Браузер обращается
только к `/api/chat`, поэтому секрет остаётся на сервере. После подключения backend
задайте переменную GitHub `VITE_CHAT_ENABLED=true`.

Сервер для публичного чата подготовлен в `render.yaml`. Развернуть бесплатную
тестовую версию можно через [Deploy to Render](https://render.com/deploy?repo=https://github.com/kassenkhann-cell/uct_sdu).
При создании Render попросит только секрет `LLM_API_KEY`. Бесплатный сервер засыпает
после простоя; для постоянного мгновенного ответа выберите платный тариф или
внутренний постоянно работающий сервер.

Либо из терминала:

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`. Команда собирает production-версию и запускает
локальный Node-сервер с API. Перед запуском она автоматически:

1. читает исходные XLSX через `xlsx`;
2. читает нестандартный CSV Е-өтініш через `papaparse`;
3. фильтрует только интернет и связь;
4. создаёт нормализованные CSV в `data/derived/`;
5. формирует локальный API-снимок для Vite.

Для production-сборки:

```bash
npm run build
```

Для запуска React и FastAPI в режиме разработки:

```bash
python -m pip install -r backend/requirements.txt
npm run dev:full
```

FastAPI: `http://localhost:8000`, документация: `http://localhost:8000/api/docs`.

## Возможности

- вкладки «Обзор» с картой, «Проблемные НП», «Обращения»,
  «Рейтинг районов», «План действий»;
- отдельная вкладка «Проблемные НП» и кликабельный KPI `Проблемные СНП — 40`;
- 311 населённых пунктов с координатами, покрытием и операторами;
- карта Leaflet с отдельным красным слоем проблемных СНП;
- переключаемые слои карты: СНП, проблемные СНП, АМС, обращения,
  ВОЛС/МШПД и высокий риск;
- 326 АМС из новой районной книги, из них 261 объект нанесён
  отдельными точками по координатам;
- сопоставление обращений с КАТО и районами;
- статус «Критический риск», если проблемный СНП совпал с профильным обращением;
- каскадные фильтры по району, населённому пункту, оператору, риску и поиску; период обращений находится только во вкладке «Обращения»;
- рейтинг районов и адресный список проблемных СНП;
- адресный план действий без внешних сервисов;
- светлая и тёмная темы, по умолчанию тёмная;
- fallback mock data при повреждении или отсутствии части источников.

## Фильтрация Е-өтініш

В расчёты входят обращения, связанные с:

- мобильной и сотовой связью;
- мобильным и фиксированным интернетом;
- низкой скоростью и качеством услуг связи;
- телекоммуникациями, покрытием и обеспечением СНП связью.

Исключаются eGov, электронное правительство, информационная безопасность, персональные данные, информационные системы, интернет-ресурсы и общая цифровизация. После профильной фильтрации и удаления повторов в текущий срез вошло 2 102 обращения.

## Источники и ClickHouse-контракт

| Исходный файл | Нормализованный источник | Назначение |
|---|---|---|
| `data/Копия МШПД в СНП 2025- последний ).xlsx` | `gold.connectivity_points` | СНП, координаты, население, охват, операторы, ВОЛС, спутник |
| `data/новые данные_по_всем_районам.xlsx` | `gold.connectivity_points` | основной источник по 12 районам: СНП, покрытие, операторы, АМС, балансодержатели и электропитание |
| `data/районам по 213v2.xlsx` | архивный источник для сравнения | прежний свод по АМС; в текущие расчёты не подмешивается |
| `data/eobr_internet_aktob_2025_2026.csv` | `gold.internet_appeals` | профильные обращения по интернету и связи |
| лист `общий` основной книги | `gold.district_connectivity` | районные итоги и планы |

В `sql/` находятся SELECT-запросы, соответствующие KPI, карте, графикам и таблицам. Все таблицы указаны как полные имена `schema.table`.

### Как извлекаются 40 проблемных СНП

Проверены все листы основной книги:

- `последний по 311`;
- `311 СНП`;
- `40 снп проблема `;
- `общий`;
- `Лист3`.

На листе `40 снп проблема ` сохранены все строки, а приоритетный список задан
оформлением ячеек и цветовым фильтром. Импорт читает сырые стили XLSX, находит
красные стили и связанные с ними стили колонки «Нас пункт», после чего получает
ровно 40 КАТО. Эти записи имеют отдельные поля проблемы, оператора, ВОЛС,
спутника и рекомендации.

## Индекс риска

Для СНП учитываются доступная технология, число операторов, наличие 4G, ВОЛС/МШПД, спутниковый канал, планы подключения, население и обращения.

Наличие АМС уменьшает инфраструктурный риск СНП. Отсутствие учтённой АМС
увеличивает риск; дополнительно учитывается спутниковая зависимость.

Районный индекс:

- 60% — инфраструктурный риск СНП;
- 25% — профильные обращения на 10 тыс. жителей;
- 15% — доля СНП высокого риска.
- дополнительный штраф — до 35 пунктов за проблемные СНП и до 15 пунктов за
  критические совпадения с обращениями.

Это внутренний управленческий индикатор, а не официальный отраслевой норматив.

## Аудит новой районной книги

- найдено и использовано 12 районных листов;
- прочитано 378 строк, включая отдельные строки для нескольких АМС одного СНП;
- сформировано 311 уникальных СНП по КАТО;
- 260 СНП содержат сведения об АМС;
- суммарно найдено 326 АМС;
- для 261 объекта удалось выделить отдельные координаты;
- использованы КАТО, район, СНП, количество и высота АМС, координаты,
  балансодержатель, электропитание, источник финансирования и признаки
  операторов;
- проигнорированы область как константа и дублирующее поле суммы строительства
  без единицы измерения.

Ранее показатель АМС был равен нулю, потому что импорт начинался до фактической
строки заголовков. Колонки читались как `__EMPTY`, поэтому КАТО и
`АМС (количество)` не связывались. Теперь строка заголовков определяется
автоматически, а объекты связываются с СНП по КАТО.

## Docker + ClickHouse

Сначала сформируйте нормализованные данные:

```bash
npm run prepare:data
docker compose up --build
```

После запуска приложение доступно на `http://localhost:8000`. Один FastAPI-контейнер обслуживает и `/api/*`, и собранный React. Локальный ClickHouse инициализируется из `data/derived/`.

Экспорт образа для передачи в контур:

```bash
docker compose build app
docker save -o image/digital-radar-aktobe-app_2026-06-23.tar digital-radar-aktobe-app:latest
```

Параметры подключения задаются через `CLICKHOUSE_HOST`, `CLICKHOUSE_PORT`, `CLICKHOUSE_USER`, `CLICKHOUSE_PASSWORD`, `CLICKHOUSE_SECURE`, `CLICKHOUSE_VERIFY`.

## Структура

```text
backend/          FastAPI и ClickHouse-режим
data/             исходные и нормализованные данные
docker/           локальная инициализация ClickHouse
frontend/         React + Vite + TypeScript + TailwindCSS
image/            шаблон окружения и экспортируемый app image
scripts/          импорт и диагностика исходных файлов
sql/              SELECT-запросы визуализаций
```

## Допущения

- основной лист СНП выбирается по слову `последний`, дополнительный — по `213`;
- колонки определяются по фактическим названиям из файлов;
- обращения уровня области учитываются в общем KPI, но не создают отдельный «район»;
- для города Актобе в инфраструктурных книгах нет сопоставимого набора СНП, поэтому полнота данных помечается как требующая уточнения;
- внутренний контур должен предоставлять таблицы или совместимые представления с именами `gold.connectivity_points`, `gold.internet_appeals`, `gold.district_connectivity`.
- 45 АМС не имеют отдельных пригодных координат и поэтому учитываются в KPI,
  районном рейтинге и карточках СНП, но не выводятся отдельным маркером;
- карточки управленческих поручений показывают рабочий сценарий и
  пока не отправляют поручения во внешнюю систему документооборота.
